Architecture

System design, module breakdown & design patterns

1. Project Overview

The java-proxy project is a high-performance, multi-threaded HTTP/HTTPS proxy server implemented in Java. Its primary purpose is to act as an intermediary for client requests, providing features such as domain-based filtering (blacklisting), user authentication via Proxy-Authorization headers, and detailed traffic logging.

The server supports standard HTTP request forwarding and HTTPS tunneling using the CONNECT method. A notable feature is its use of modern Java concurrency primitives to handle multiple simultaneous connections efficiently.

2. Technology Stack

  • Language: Java (Version 21+ recommended due to the use of Virtual Threads).
  • Concurrency: Project Loom / Virtual Threads (Executors.newVirtualThreadPerTaskExecutor()).
  • I/O: Standard Blocking I/O (java.net.Socket, java.io) with stream piping.
  • Configuration: Java Properties API and File NIO.

3. Architecture Style

The project follows a Layered Monolith architecture with a focus on Interface-based Programming in the data source layer. The architecture is divided into clear functional areas:

  • Entry Point: Orchestrates the server socket and thread pool.
  • Service Layer: Contains the core business logic for protocol handling and logging.
  • Data Source Layer: Abstracted via interfaces to allow multiple implementations for retrieving blocked domains and user credentials.
  • Helper/Utility Layer: Provides cross-cutting concerns like configuration loading and string matching.

6. Key Design Patterns

  • Factory Pattern: BlockedDomainFactory encapsulates the logic for aggregating blocked domains from multiple sources (Properties and Files), shielding the ProxyHandler from data retrieval details.
  • Strategy Pattern: The use of BlockedDomains and Users interfaces allows the application to swap data sources (e.g., moving from a flat file to a database) without changing the service logic.
  • Singleton/Static Utility: AppConfig acts as a centralized, globally accessible configuration provider initialized via a static block.
  • Worker Thread (Command): ProxyHandler implements Runnable, encapsulating the "command" of processing a socket connection, which is then executed by the ExecutorService.

7. Configuration & Infrastructure

Configuration Files

The application relies on external resources located in a Resources/ directory:

  • config.properties: Contains system settings like app.port, remote.timeout, and buffer.size.
  • user.properties: Stores user credentials in username=password format.
  • light.txt: A newline-separated list of blocked domains.

Infrastructure Notes

  • Concurrency Model: By using Executors.newVirtualThreadPerTaskExecutor(), the server avoids the overhead of traditional thread pools, allowing it to scale to thousands of concurrent connections with minimal memory footprint.
  • Error Handling: The system uses a "Quiet Close" mechanism and 502 Bad Gateway responses to ensure that remote failures do not crash the proxy service.
  • Logging: Synchronized file writing ensures thread safety when multiple virtual threads attempt to log activity to proxy.log.

API Specification

Endpoints, request/response models & authentication

1. API Overview

The java-proxy is a high-performance transparent forward proxy server. It intercepts client requests, performs domain-based filtering, validates user credentials, and forwards traffic to destination servers.

  • Base URL: http://<proxy-host>:<port> (Default port: 8080)
  • Authentication: Proxy-Authorization (Basic scheme: Base64 encoded username:password)
  • Content Types: Transparent (supports all media types handled by the target servers)
  • Protocol Support: HTTP/1.1 and HTTPS (via CONNECT tunneling)

2. Endpoints Table

As a forward proxy, this application does not host specific REST resources but acts as a gateway for all outbound traffic.

Method Path Description Request Body Response HTTP Status Codes
CONNECT {host}:{port} Establishes an HTTPS tunnel to the destination server. Encrypted Stream Binary Stream 200, 502
GET, POST, PUT, etc. {absolute_uri} Proxies standard HTTP requests to the target host. Varies by request Varies by target 200, 502

4. Error Handling

The proxy returns standard HTTP status codes when it encounters issues connecting to the upstream server or validating the request.

Code Meaning Description
502 Bad Gateway Remote Connection Failed The proxy could not establish a connection to the destination host or the remote host timed out.
Connection Closed Unauthorized / Blocked If the domain is in the blocklist or credentials are invalid, the proxy logs the event and terminates the socket connection.

5. Security

Authentication

The proxy implements Proxy Authentication. Clients must provide the following header:

Proxy-Authorization: Basic <Base64(username:password)>

Domain Filtering

The server implements a multi-source blocklist strategy:

  • File-based: Domains listed in Resources/light.txt.
  • Property-based: Keywords defined in config.properties under blocked.websites.keyword.
  • Subdomain Matching: The LinkMatcher recursively checks subdomains (e.g., if example.com is blocked, sub.example.com is also blocked).

6. Endpoint Details

HTTPS Tunneling (CONNECT)

Used by clients to initiate SSL/TLS communication through the proxy.

Parameters

  • Header: Proxy-Authorization (Required)
  • Target: The destination host and port (e.g., google.com:443)

Example Request

CONNECT google.com:443 HTTP/1.1
Proxy-Authorization: Basic dXNlcjpwYXNzd29yZA==

Example Response

HTTP/1.1 200 Connection Established

HTTP Proxying

Forwards standard HTTP requests. The proxy rewrites the request line to use the relative path before forwarding to the remote host.

Request Body

The proxy supports request bodies (e.g., for POST) by reading the Content-Length header and streaming the specified number of bytes to the remote server.

Example Request

GET http://example.com/index.html HTTP/1.1
Host: example.com
Proxy-Authorization: Basic dXNlcjpwYXNzd29yZA==

Example Response

Content-Type: text/html
